# **Decision Guidance**

This directory contains guidance to help teams determine whether they need to create an RFC, ADR, or Standard. It provides clarity on the purpose of each artifact and when to use them.

### **Purpose of Decision Guidance**
- Help teams choose the right artifact for their needs.
- Provide clear criteria for creating RFCs, ADRs, or Standards.
- Reduce confusion and ensure consistent decision-making.

### **How to Use This Directory**
1. **Read Guidance**: Refer to the following documents to determine the appropriate artifact:
   - [Do I Need an RFC?](do-i-need-a-rfc.md)
   - [Do I Need an ADR?](do-i-need-an-adr.md)
   - [Do I Need a Standard?](do-i-need-a-standard.md)
2. **Follow Recommendations**: Use the templates provided in the respective directories to create your artifact.

### **Key Files**
- [Do I Need an RFC?](do-i-need-a-rfc.md): Guidance for creating RFCs.
- [Do I Need an ADR?](do-i-need-an-adr.md): Guidance for creating ADRs.
- [Do I Need a Standard?](do-i-need-a-standard.md): Guidance for creating Standards.
